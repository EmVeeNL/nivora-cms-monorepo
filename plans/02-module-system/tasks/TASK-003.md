## Summary
Update `src/router.tsx` to import `adminRouteFactories` from the generated registry and assemble the full TanStack Router route tree by combining existing file-based core routes with programmatic module admin routes. Define the `AdminRouteFactory` type contract that all module `routes/admin/index.ts` files must export.

## Business Value
Without this wiring, module admin routes never appear in the router. This is the bridge between the module system and the app's navigation ‚Äî every new module gets its admin pages registered here automatically, with no manual changes to `src/router.tsx`.

## Acceptance Criteria
- [ ] `src/router.tsx` assembles the route tree from both file-based core routes and `adminRouteFactories`
- [ ] Adding a new `adminRouteFactory` to the generated registry automatically makes its route navigable
- [ ] The `_app` pathless layout (`src/routes/_app.tsx`) remains the parent for all admin routes
- [ ] Module routes use `lazyRouteComponent` for code splitting
- [ ] TypeScript compiles cleanly ‚Äî no `any`, no `@ts-ignore`
- [ ] `pnpm lint` and `pnpm build` pass

## Requirements
- Each module's `routes/admin/index.ts` must export `adminRouteFactory: AdminRouteFactory`
- `AdminRouteFactory` is typed in `src/core/module/types.ts`
- Route factories receive the `_app` route as their parent
- Module routes are lazy-loaded (dynamic import of the component)
- `routeTree.gen.ts` continues to provide TypeScript route type augmentations ‚Äî it is NOT removed

## Technical Design

### `AdminRouteFactory` type (add to `src/core/module/types.ts`)
```ts
import type { AnyRoute, RouteOptions } from '@tanstack/react-router'

export type AdminRouteFactory = (parentRoute: AnyRoute) => {
  addChildren: (children?: unknown[]) => unknown
  // The actual return is a TanStack RouteDefinition;
  // typed loosely here because programmatic routes don't participate
  // in the routeTree.gen.ts type system
}
```

Or more practically, just:
```ts
export type AdminRouteFactory = (parentRoute: AnyRoute) => AnyRoute
```

### Module `routes/admin/index.ts` shape (convention)
```ts
// src/modules/core/routes/admin/index.ts
import { createRoute, lazyRouteComponent } from '@tanstack/react-router'
import type { AnyRoute } from '@tanstack/react-router'

export const adminRouteFactory = (parentRoute: AnyRoute) =>
  createRoute({
    getParentRoute: () => parentRoute,
    path: '/',
    component: lazyRouteComponent(() => import('./DashboardPage')),
  })
```

Note: The core module's admin route at `path: '/'` replaces (and migrates from) `src/routes/_app/index.tsx`. See TASK-005.

### Updated `src/router.tsx`

The current `router.tsx` imports `routeTree` directly from `routeTree.gen.ts`. We update it to:

1. Import the raw route definitions from their files (not the assembled tree from `routeTree.gen.ts`)
2. Import `adminRouteFactories` from the generated registry
3. Assemble the tree manually

```ts
import { createRouter as createTanStackRouter } from '@tanstack/react-router'
// Raw route definitions (not the assembled routeTree)
import { Route as rootRoute } from './routes/__root'
import { Route as AppRoute } from './routes/_app'
// Generated module registry
import { adminRouteFactories } from '#/core/module-registry.gen.ts'

export function getRouter() {
  // Build module routes with AppRoute as their parent
  const moduleRoutes = adminRouteFactories.map((factory) => factory(AppRoute))

  // Assemble the route tree manually
  const routeTree = rootRoute.addChildren([
    AppRoute.addChildren(moduleRoutes),
  ])

  return createTanStackRouter({
    routeTree,
    scrollRestoration: true,
    defaultPreload: 'intent',
    defaultPreloadStaleTime: 0,
  })
}

declare module '@tanstack/react-router' {
  interface Register {
    router: ReturnType<typeof getRouter>
  }
}
```

### Impact on `routeTree.gen.ts`
`routeTree.gen.ts` is still generated by `@tanstack/router-plugin` but is no longer imported for tree assembly. Its `declare module '@tanstack/react-router'` augmentations remain useful for type completions on `useNavigate`, `Link`, etc. for any remaining file-based routes.

When the core module's admin route migrates out of `src/routes/_app/index.tsx` (TASK-005), TanStack Router's scanner will regenerate `routeTree.gen.ts` without that route. The file-based route tree may end up empty except for `__root` and `_app` ‚Äî that is expected and fine.

### Route path strategy for modules
Each module declares its admin path in `routes/admin/index.ts` via the `path` option in `createRoute`. Convention:
- Core module: `path: '/'` (the dashboard home)
- Other modules: `path: '/[module-slug]'` e.g. `path: '/blog'`

This means the full admin URL for a blog module would be `/blog` (under the `_app` pathless layout which is at `/`).

## Instructions for Developer
- When importing raw route definitions from `./routes/__root` and `./routes/_app`, import the named `Route` export ‚Äî both files use `createFileRoute` which returns a route builder; the `.update({ getParentRoute })` call in `routeTree.gen.ts` was needed for the assembled tree, but when we call `.addChildren` directly on the imported route, TanStack handles the parent relationship automatically.
- If TypeScript complains about the `addChildren` return type being incompatible, cast to `AnyRoute[]` for the `moduleRoutes` array. The type system for mixed file+programmatic trees is not perfect; functional correctness matters more than type perfection here.
- Keep the `declare module '@tanstack/react-router'` block in `router.tsx` ‚Äî removing it breaks `useNavigate` and `Link` type inference.
- Do NOT delete `routeTree.gen.ts` or remove `@tanstack/router-plugin` from `vite.config.ts` ‚Äî the plugin still runs and produces type augmentations.
